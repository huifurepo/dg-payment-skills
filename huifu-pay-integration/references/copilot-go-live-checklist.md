## 通用上线检查

| 检查项 | 要求 |
| --- | --- |
| 商户和系统标识 | `huifu_id`、`sys_id`、`product_id` 已确认 |
| 托管项目 | 托管场景的 `project_id` 已确认且项目状态可交易 |
| 密钥 | 生产密钥只放服务端安全环境，不入仓库、不入前端 |
| 回调地址 | `notify_url` HTTPS、公网可达、无错误重定向、回包符合要求 |
| 异步通知 | 已完成验签、幂等、固定成功应答、失败重试监控 |
| 查单兜底 | 支付终态必须有异步通知验签/幂等和查单二次确认或补偿查询闭环 |
| 退款 | 退款申请、退款查询、退款幂等已完成 |
| 对账 | 对账文件或交易查询链路已确认 |
| 日志 | 敏感字段脱敏，保留请求流水和平台流水 |
| 前端 | callback 不直接改订单成功 |

## 存量系统上线检查

- 明确新增了哪些配置、服务、适配层和回调处理。
- 明确哪些原逻辑被修改，哪些保持不动。
- 完成订单状态映射和异常状态兜底。
- 完成重复通知幂等保护。
- 保存支付和退款关键流水。
- 支付成功后的发货、开通权益、发券等动作不会重复执行。
- 退款成功后的库存、权益、账务处理有明确责任边界。
- 有灰度策略、回滚策略和生产监控。

## 必测项

| 场景 | 必测内容 |
| --- | --- |
| 正常下单/预下单 | 请求成功、返回字段落库、用户可支付或跳转 |
| 参数缺失 | 本地校验能拦截必填和条件必填 |
| 支付成功异步通知 | 验签、解析、状态更新、固定回包 |
| 重复异步通知 | 不重复发货、不重复改状态 |
| 查单确认 | 查单能确认最终状态，处理中可重试 |
| 全额/部分退款 | 退款申请、退款查询、幂等 |
| 分账参数 | 分账接收方、金额/比例、权限检查 |
| checkout-js | 前端 callback 后由服务端确认 |
| 存量状态流转 | 新旧渠道状态不互相破坏 |
| 灰度和回滚 | 可切换路由，可停止新交易入口 |

## 升级人工材料

生产问题只整理材料：

- 环境：联调/生产、产品线、接口名、技术栈、SDK 版本。
- 标识：脱敏后的 `huifu_id`、`sys_id`、`product_id`、`project_id`。
- 流水：`req_date`、`req_seq_id`、`hf_seq_id`、退款流水。
- 时间：请求时间、回调时间、查单时间。
- 报文：脱敏请求、响应、异步通知原文、验签结果。
- 影响：交易笔数、金额范围、业务影响、是否可重试。
- 已做动作：重试、查单、关单、退款、人工处理记录。

## 拒答边界

- 不回答费率、合规、政策审批、通道准入结论。
- 不提供绕过验签、绕过风控、伪造成功、跳过查单的代码。
- 不对生产事故定责。
- 不保存或复述真实密钥和真实用户数据。
