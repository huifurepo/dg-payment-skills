# 凭据使用规则与存放边界

本文档适用于聚合支付、托管支付和商户进件三条业务文档链路。

## 凭据角色说明

当前 Skill 文档中列出的 `HUIFU_PRODUCT_ID`、`HUIFU_SYS_ID`、`HUIFU_RSA_PRIVATE_KEY`、`HUIFU_RSA_PUBLIC_KEY` 等运行时配置项，用于让**接入方应用**在运行时完成汇付 SDK 初始化，并在应用侧完成请求签名和响应验签。

## 存放边界

- 本仓库内的接入文档与模板只声明接入方应用需要准备哪些配置
- Skill 包内不会保存、落库、缓存、回传、打印或持久化任何商户凭据
- 安装或阅读 Skill 不会触发任何凭据采集、网络上传、文件写入或本地持久化动作
- 真实使用发生在接入方应用代码运行时

## 接入方应做到

- 凭据应由接入方项目的环境变量、CI/CD Secret 或密钥管理系统持有
- 开发和联调阶段务必使用联调专用密钥，不要提供生产密钥
- 日志中避免打印完整密钥信息
- RSA 私钥切勿上传到代码仓库
- 上线前应确认生产密钥只存在于受控运行环境中

## 商户进件敏感资料

身份证/营业执照号码、银行卡号、手机号、地址、材料图片及其 URL、`file_id`、`token_no`、实名二维码和电子签约链接都属于高敏感数据：

- 默认不记录原文、不写入 Skill、回归 fixture、日志或前端响应
- 确有服务端持久化必要时，采用字段级加密、最小权限、访问审计与明确保留期
- 日志仅保留掩码或不可逆摘要；任何联调样本均需先脱敏并移除密钥和生产流水

## 语言适配说明

当前仓库内的完整代码示例以 Java 和 PHP 为主，其他语言先通过基础文档中的语言适配入口接入。详见 `references/shared-server-sdk-matrix.md`。

## 运行时配置说明

这些配置项当前通过 `SKILL.md` 正文、`references/` 文档和接入示例暴露，不依赖非官方 `metadata` / `config` 字段。

文档维护原则：

- 运行时配置项只在正文和 reference 中维护一次，避免多处结构化字段漂移
- 如果后续平台恢复官方配置声明能力，应以平台正式规范为准，再决定是否引入结构化配置块
