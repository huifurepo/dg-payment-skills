# 前端 SDK 能力矩阵

当前前端能力定位是：商户自定义收银台页面 + 内嵌支付组件。

## 当前入口

| 项目 | 说明 |
| --- | --- |
| 对应文档 | `references/checkout-js.md` |
| npm 包 | `@dg-elements/js-sdk` |
| 版本策略 | 接入时以项目锁定版本为准；升级前查询 npm registry，不在 Skill 中声称“最新版” |
| 页面归属 | 商户自有页面 |
| 组件能力 | `checkout`、支付宝按钮、微信按钮、云闪付按钮 |
| 最终结果确认 | 服务端闭环；异步通知需验签、幂等、状态锁，并结合查单二次确认或补偿查询 |

## 前端负责什么

- 调用自有服务端 `createPreOrder`
- 初始化 `HFPay`
- 渲染 `checkout` 或单支付按钮
- 监听前端回调并驱动页面状态

## 前端不负责什么

- 不持有商户私钥
- 不直接请求汇付签名网关
- 不判定最终支付成功
- 不替代服务端查询和异步通知

## 环境展示规则

- 支付宝内：只展示支付宝
- 微信内：只展示微信
- 云闪付内：只展示云闪付
- 系统浏览器：默认展示支付宝和微信

## 当前覆盖范围

`1.3.0` 版本当前覆盖：

- SDK 安装
- `HFPay` 初始化
- `createPreOrder` 契约
- `pre_order_type` 对应关系
- `callback` 回调处理边界

当前不覆盖：

- 固定前端工程模板
- React / Vue / 原生 JS 三套完整脚手架
- 自动生成前端项目代码

## 与服务端的协作链路

```text
hostingpay-preorder
  -> checkout-js
  -> shared-async-notify
  -> hostingpay-query
```

也就是说，前端组件接入必须与服务端预下单和服务端查单一起看，不能独立理解成“只接 JS SDK 就能完成支付闭环”。
